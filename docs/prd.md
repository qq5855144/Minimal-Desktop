# 需求文档

## 1. 应用概述

### 1.1 应用名称
iOS风格桌面管理系统

### 1.2 应用描述
一个模仿iOS手机系统桌面的网页应用，支持应用图标管理、拖拽排序、文件夹组织、多页切换以及GitHub数据同步功能。整体采用iOS风格设计，包括毛玻璃效果、圆角图标、底部Dock栏、时钟Widget、搜索框Widget等视觉元素。

## 2. 用户与使用场景

### 2.1 目标用户
需要在网页端管理常用网站链接的用户，希望通过类似iOS桌面的方式组织和访问网站。

### 2.2 核心使用场景
- 用户添加常用网站到桌面，通过点击图标快速访问
- 用户通过拖拽方式整理应用位置和创建文件夹
- 用户在多个桌面页面间切换管理不同类别的应用
- 用户通过GitHub同步功能在不同设备间同步桌面配置
- 用户通过搜索框快速访问网站或搜索内容

## 3. 页面结构与功能说明

### 3.1 页面结构

```
├── 主界面（桌面）
│   ├── 时钟Widget
│   ├── 搜索框Widget
│   ├── 应用图标区域
│   ├── 文件夹
│   ├── 系统应用图标（添加应用、设置、同步）
│   ├── 底部Dock栏
│   └── 页面指示器
├── 添加应用弹窗
├── 编辑应用弹窗
├── 应用上下文菜单
├── 文件夹展开视图
├── 应用设置页面
└── 数据同步页面
```

### 3.2 主界面（桌面）

#### 3.2.1 时钟Widget
- 位于桌面顶部
- 显示实时时钟（时:分格式）
- 显示日期、星期、农历信息
- 支持拖拽与搜索框Widget交换位置

#### 3.2.2 搜索框Widget
- 位于桌面顶部（默认在时钟Widget下方或旁边）
- 采用毛玻璃效果
- 用户输入内容后自动判断为URL或搜索词
- 若为URL则直接打开对应网站
- 若为搜索词则使用默认搜索引擎搜索
- 支持拖拽与时钟Widget交换位置

#### 3.2.3 应用图标区域
- 展示用户添加的应用图标
- 应用图标采用圆角设计
- 图标下方显示应用名称
- 点击图标打开对应网站URL
- 长按图标弹出上下文菜单（包含「编辑」和「删除」选项）
- 加载时显示骨架屏效果（网格布局占位）

#### 3.2.4 文件夹
- 文件夹图标显示内部应用的缩略图（最多显示9个）
- 文件夹下方显示文件夹名称
- 点击文件夹展开文件夹视图
- 文件夹内容加载时显示骨架屏效果

#### 3.2.5 系统应用图标
- **「添加应用」图标**：固定在桌面，不可删除，点击打开添加应用弹窗
- **「设置」图标**：固定在桌面，不可删除，点击进入应用设置页面
- **「同步」图标**：固定在桌面，不可删除,点击进入数据同步页面

#### 3.2.6 底部Dock栏
- 固定在页面底部
- 可通过拖拽将应用图标放入Dock栏
- 可通过拖拽将Dock栏内图标取出放回主界面
- Dock栏内图标支持长按弹出上下文菜单
- 采用毛玻璃效果

#### 3.2.7 页面指示器
- 显示当前所在页面位置
- 显示总页面数量
- 支持点击切换页面

### 3.3 添加应用弹窗

- 输入网站URL
- 自动抓取网站favicon作为图标
- 支持手动上传自定义图标
- 输入应用名称
- 支持自定义应用颜色
- 确认添加后将应用图标放置到桌面

### 3.4 编辑应用弹窗

- 修改应用名称
- 修改应用URL
- 更换应用图标（重新抓取favicon或上传自定义图标）
- 修改应用颜色

### 3.5 应用上下文菜单

- 长按应用图标触发
- 包含「编辑」选项：点击打开编辑应用弹窗
- 包含「删除」选项：点击删除该应用

### 3.6 文件夹展开视图

- 展示文件夹内所有应用图标（最多9个）
- 显示文件夹名称，支持点击重命名
- 文件夹内应用支持拖拽排序
- 支持将应用拖拽到文件夹外放置到主界面
- 点击空白区域或返回按钮关闭文件夹视图
- 加载时显示骨架屏效果

### 3.7 应用设置页面

- 提供应用相关设置选项

### 3.8 数据同步页面

- 输入GitHub Token进行登录/认证
- 选择目标GitHub仓库
- 提供「同步到GitHub」按钮，将当前桌面数据上传到GitHub仓库
- 提供「从GitHub同步」按钮，从GitHub仓库下载数据并覆盖本地桌面配置
- 显示同步状态和结果

## 4. 业务规则与逻辑

### 4.1 骨架屏加载规则

- 主界面应用图标网格加载时显示骨架屏占位
- 文件夹内容加载时显示骨架屏占位
- 骨架屏样式与实际内容布局一致

### 4.2 分页切换规则

- 将应用拖拽到桌面左边缘悬停时触发向上一页翻页
- 将应用拖拽到桌面右边缘悬停时触发向下一页翻页
- 悬停时间超过设定阈值后自动切换页面
- 第一页时左边缘不触发翻页，最后一页时右边缘不触发翻页
- 支持通过页面指示器点击切换页面
- 页面指示器实时显示当前所在页面

### 4.3 添加应用规则

- 用户点击桌面上的「添加应用」系统图标，打开添加应用弹窗
- 用户输入网站URL后，系统自动抓取该网站的favicon作为图标
- 若自动抓取失败或用户选择自定义，支持手动上传图标
- 应用名称默认为网站标题，支持用户自定义修改
- 支持自定义应用颜色
- 新添加的应用默认放置在当前页面的空白位置

### 4.4 编辑与删除应用规则

- 长按应用图标弹出上下文菜单
- 上下文菜单包含「编辑」和「删除」选项
- 点击「编辑」打开编辑应用弹窗，可修改应用名称、URL、图标、颜色
- 点击「删除」删除该应用，该位置变为空白

### 4.5 文件夹创建规则

- 将一个应用拖拽到另一个应用上悬停600ms后自动合并为文件夹
- 文件夹默认名称为「文件夹」，支持重命名
- 文件夹内最多放置9个应用
- 当文件夹内只有1个应用时，文件夹自动解散，应用回到主界面原位置

### 4.6 文件夹管理规则

- 支持重命名文件夹
- 文件夹内应用支持拖拽排序
- 支持将文件夹内应用拖拽到主界面
- 文件夹内应用数量达到9个时，无法再添加新应用

### 4.7 拖拽排序规则

- 采用绝对放置逻辑，不依赖紧凑布局
- 拖拽应用到目标位置时，如果目标位置已有应用则交换位置
- 拖拽应用到空白位置时，应用放置在该位置
- 拖拽应用到桌面边缘悬停时触发翻页，翻页后继续拖拽操作

### 4.8 搜索框规则

- 用户在搜索框输入内容后，系统自动判断输入内容为URL或搜索词
- 若输入内容符合URL格式（如包含http://、https://或域名格式），则直接打开对应网站
- 若输入内容为普通文本，则使用默认搜索引擎进行搜索

### 4.9 Widget拖拽规则

- 时钟Widget和搜索框Widget支持拖拽交换位置
- 拖拽时显示目标位置预览

### 4.10 Dock栏规则

- 用户可将主界面应用图标拖拽到Dock栏放置
- 用户可将Dock栏内图标拖拽取出放回主界面
- Dock栏内图标支持长按弹出上下文菜单（包含「编辑」和「删除」选项）

### 4.11 数据同步规则

- 用户使用GitHub Token登录/认证后，选择目标仓库
- 同步到GitHub时，将当前桌面配置数据（应用列表、文件夹结构、位置信息、Widget配置等）上传到指定仓库
- 从GitHub同步时，从指定仓库下载数据并覆盖本地桌面配置
- 同步操作需显示进度和结果反馈

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 输入的URL无法访问或无效 | 提示用户URL无效，要求重新输入 |
| 无法抓取网站favicon | 提示用户手动上传图标或使用默认图标 |
| 文件夹内应用数量已达9个 | 禁止继续添加应用到该文件夹，提示用户文件夹已满 |
| 拖拽应用到文件夹时文件夹已满 | 不执行合并操作，应用回到原位置 |
| 拖拽应用到另一应用上悬停时间不足600ms | 不触发文件夹合并，应用回到原位置 |
| GitHub Token无效或过期 | 提示用户Token无效，要求重新输入 |
| GitHub仓库不存在或无权限 | 提示用户仓库不存在或无权限，要求重新选择 |
| 同步过程中网络中断 | 提示同步失败，保留本地数据不变 |
| 从GitHub同步的数据格式错误 | 提示数据格式错误，不覆盖本地数据 |
| 搜索框输入为空 | 不执行任何操作 |
| 尝试删除系统应用图标 | 禁止删除操作，提示系统应用不可删除 |

## 6. 验收标准

1. 用户打开应用，看到iOS风格的桌面界面，包括时钟Widget、搜索框Widget、应用图标、系统应用图标（添加应用、设置、同步）、底部Dock栏、页面指示器
2. 用户点击「添加应用」系统图标，打开添加应用弹窗，输入网站URL和名称，确认后应用图标出现在桌面
3. 用户长按应用图标，弹出上下文菜单，点击「编辑」，修改应用名称和图标，确认后应用信息更新
4. 用户拖拽一个应用到另一个应用上悬停600ms，两个应用合并为文件夹，文件夹名称显示为「文件夹」
5. 用户点击文件夹，文件夹展开，显示内部应用，用户拖拽应用调整顺序，关闭文件夹后顺序保持
6. 用户拖拽应用到桌面右边缘悬停，桌面自动切换到下一页，用户将应用放置在新页面
7. 用户在搜索框输入网站URL，按回车后直接打开对应网站
8. 用户点击「同步」系统图标，输入GitHub Token，选择仓库，点击「同步到GitHub」，提示同步成功
